Introduction:¶

(Explain Anime) (Tell how popular it is.) Anime is animation that originates from Japan. It has a distinct art/drawing and/or computer animation style with Japanese voice over. In addition, there are versions of the same animation with Japanese voice overs and subtitles in various languages. Also, other voice overs exist in other languages. Although originally intended for propaganda, Anime evolved and was introduced to the world through various films and series. Today, Anime continues to grow in popularity and is often aired on international networks.

(Motivation) There is much debate about what makes successful anime (link to reddit pages). Usually the main explanations are those of plot, character development, complexity... (etc). Often times we hear that certain anime are overrated, underrated, or striving because of the past success of certain authors or studios. Which begs the question: What about the outside factors that impact the success of an anime? The motivation behind this walkthrough is to determine if the positively or negatively impact the rating that an anime receives from the public. Here, successs is based on the rating given and the factors into play are focused on the display and setup that is a part of the airing process instead of focusing on the content of the anime itself.

(Reference the website) (Explain the website) The dataset in use can be found from kaggle: https://www.kaggle.com/canggih/anime-data-score-staff-synopsis-and-genre The data from the dataset came from a very active anime and manga social networking website: https://myanimelist.net/ The webiste is used to catalog anime and manga, share details, and give scores based on user ratings.

This walkthrough goes through the data pipline: Data collection Data Management Exploratory Analysis Hypothesis Testinga and Machine Learning Conclusions

By the end of this walkthrough you will be able to alter data into a comfortable format, apply descriptive statistics, visualize data, determine the existence of trends, and create a machine learning model to predict scores and classify anime.

Data Management / Representation¶

We will alter the table so that any missing data will have 'Nan' which is a commmon data type to represent uncommon/nonexistent/undefined values. We do this so that the processing of data is simpler and does not produce errors. Furthermore, we will remove attributes from the relation that are not required for our analysis.

This is also where we can add more columns and replace existent values for ease of future computation.

The numpy libary will be used to replace values in the table.

Here Pearson's standard correlation coefficient is used. In general this is a fast way to see if there is a correlation between your variables. However, it only works for numerical variables. Looking at our dataset, we have categorical variables to take into account.

A common method to account for categorical variables is to create dummy variables and conduct a regression. Since there are 42 genres, there would be 41 dummy variables (1 would be left out as a baseline to prevent multicolinearity). The dummy variables would take up the form of all possible values according to the categorical variables. In addition, there would take on the values of 0 or 1.

If each anime was assigned 1 genre, then you could simplify this process with the ols library. Read more here: https://douglaspsteen.github.io/handling_categorical_variables_with_statsmodels_ols
