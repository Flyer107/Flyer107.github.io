Predicting Anime Ratings¶

Kwesi Owubah

Introduction:¶

Anime is animation that originates from Japan. It has a distinct art/drawing and/or computer animation style with Japanese voice over. In addition, there are versions of the same animation with Japanese voice overs and subtitles in various languages. Also, other voice overs exist in other languages. Although originally intended for propaganda, Anime evolved and was introduced to the world through various films and series. Today, Anime continues to grow in popularity and is often aired on international networks.

There is much debate about what makes successful anime (link to reddit pages). Usually the main explanations are those of plot, character development, complexity... (etc). Often times we hear that certain anime are overrated, underrated, or striving because of the past success of certain authors or studios. Which begs the question: What about the outside factors that impact the success of an anime? The motivation behind this walkthrough is to determine if the positively or negatively impact the rating that an anime receives from the public. Here, successs is based on the rating given and the factors into play are focused on the display and setup that is a part of the airing process instead of focusing on the content of the anime itself.

The dataset in use can be found from kaggle: https://www.kaggle.com/canggih/anime-data-score-staff-synopsis-and-genre The data from the dataset came from a very active anime and manga social networking website: https://myanimelist.net/ The webiste is used to catalog anime and manga, share details, and give scores based on user ratings.

This walkthrough goes through the data science pipline: Data collection Data Management Exploratory Analysis Hypothesis Testinga and Machine Learning Conclusions

By the end of this walkthrough you will be able to alter data into a comfortable format, apply descriptive statistics, visualize data, determine the existence of trends, and create a machine learning model to predict scores.

Data Management / Representation¶

We will alter the table so that any missing data will have 'Nan' which is a commmon data type to represent uncommon/nonexistent/undefined values. We do this so that the processing of data is simpler and does not produce errors. Furthermore, we will remove attributes from the relation that are not required for our analysis.

This is also where we can add more columns and replace existent values for ease of future computation.

The numpy libary will be used to replace values in the table.

Here Pearson's standard correlation coefficient is used. In general this is a fast way to see if there is a correlation between your variables. However, it only works for numerical variables. Looking at our dataset, we have categorical variables to take into account.

A common method to account for categorical variables is to create dummy variables and conduct a regression. Since there are 42 genres, there would be 41 dummy variables (1 would be left out as a baseline to prevent multicolinearity). The dummy variables would take up the form of all possible values according to the categorical variables. In addition, there would take on the values of 0 or 1.

If each anime was assigned 1 genre, then you could simplify this process with the ols library. Read more here: https://douglaspsteen.github.io/handling_categorical_variables_with_statsmodels_ols

Hypothesis Testing and Machine Learning¶

Now we will attempt to predict the score of an anime given the factors from our table. We will do so by conducting a multiple linear regression. The response variable is numerical (score) and the predictors are a mix of categorical variables (as dummy variables) and numerical variables. It is important to note that if our response variable was categorical, a method to handle this hurdle would be to conduct a logistic regression.

Here is an article that warns us when conducting multiple linear regression. There are certain pitfalls to be careful about: https://www.kaggle.com/lukewiebolt/nhl-player-salary-multiple-regression

H0: Beta1 = Beta2 = ... = BetaN = 0 Null Hypothesis: Meaning that there is no correlation between

Ha: Betak != 0 Alternate Hypothesis: Meaning that there is some variable that is correlated with a response.

The Ordinary Least Squares regression was conducted. There was a warning underneath the regression that there might be multicollinearity. Therefore, another heatmap was created to check if any of the independent variables (independent to independent, not independent to dependent) were strongly correlated with each other. The output shows that the R-Squared is 0.313. So about 31.3% of the response is explained by the predictors. In statistics R-squared over 90% is excellent. R-squared over 70% is typically acceptable. Anything underneath 40% is regarded as a low correlation. The value of the R-Squared we obtained is not that great. However, looking at the t-test in reference to the p-value (P > |t|) we see that a few of the predictors respect our confidence level of 5%.
